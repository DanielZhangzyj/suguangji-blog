# 在腾讯云平台创建并部署个人网站 —— 完整学习教程

> **适用场景**：你想在腾讯云上拥有"属于自己的网站"（自定义域名 + HTTPS），从零开始：本地写页面 → 传 GitHub → 在腾讯云开服务 → 部署上线 → 日常维护。
> **本文来源**：基于一次真实项目——把一个 Vite + React 个人博客从 Cloudflare Pages 迁到**腾讯云轻量应用服务器（Lighthouse）**，并绑定自定义域名 `danielsdailynote.cn`。所有命令都来自实操，踩过的坑也一并标出。

---

## 0. 先选一条路线

腾讯云上"放一个网站"有三条主流路线，先想清楚你要哪种：

| 路线 | 适合谁 | 自定义域名 | 证书 | 运维成本 |
|------|--------|-----------|------|---------|
| **A. 轻量应用服务器 Lighthouse + 自装 nginx**（本文主线） | 想要"自己的服务器"、要自定义域名 | ✅ 支持 | 你自备（免费 90 天） | 中（系统更新、nginx 配置） |
| **B. CloudBase 静态托管** | 只想最快上线、不在乎默认域名 | ❌ 2026 起静态托管已不支持绑自定义域名 | 默认域名自动 HTTPS | 极低 |
| **C. EdgeOne Pages** | 静态站 + 自动续期证书 | ✅ 支持 | 自动（接近免费） | 低 |

> **结论**：想用自己域名且有 HTTPS，**路线 A（Lighthouse）最稳**。路线 B 现在只能给一个 `xxx.tcloudbaseapp.com` 的默认域名（自带 HTTPS，够用但不好记）。路线 C 最省心但有连接器前置要求。
> 下面按 **路线 A** 完整走一遍。

## 1. 在本地做 HTML 项目

### 1.1 最简单：手写静态文件

建一个文件夹，放 `index.html`（网站入口**必须**叫这个）：

```
my-site/
├── index.html      # 首页（必须）
├── style.css
├── app.js
└── images/
```

直接双击 `index.html` 就能本地预览。

### 1.2 用现代框架（推荐，便于维护）

以 **Vite + React** 为例：

```bash
npm create vite@latest my-site -- --template react-ts
cd my-site
npm install
npm run dev        # 本地开发预览 http://localhost:5173
```

写完后**构建**出生产文件：

```bash
npm run build      # 产物在 dist/ 目录
```

`dist/` 里就是可以直接丢到服务器上的静态文件（`index.html` + `assets/`）。

> ⚠️ **踩坑：项目路径含 `&` 或中文，build 会失败**
> 例如目录叫 `Web&App 开发`，`npm run build` 会把 `&` 当成命令分隔符，把中文路径截断成乱码，`tsc` 找不到模块。
> **解法**：绕过 npm 脚本，直接用 node 调本地二进制：
> ```bash
> node node_modules/typescript/bin/tsc -b
> node node_modules/vite/bin/vite.js build
> ```

### 1.3 路由方式影响部署

- 若用 **HashRouter**（链接带 `#`，如 `#/article/1`）：服务器永远只收到 `/index.html`，**不需要特别配置**，最省心。
- 若用 **BrowserRouter**（干净链接 `/article/1`）：nginx 需加 `try_files $uri /index.html;` 做 history 回退，否则刷新深链接会 404。

## 2. 用 Git 把项目上传到 GitHub

GitHub 是代码的"云端备份 + 协作中心"。即使你自己部署，也建议传上去。

### 2.1 初始化并提交

```bash
cd my-site
git init                      # 初始化仓库
git add .                     # 暂存所有文件
git commit -m "first commit" # 提交
```

> 建议加一个 `.gitignore`，忽略 `node_modules/`、`dist/` 等：
> ```
> node_modules/
> dist/
> .env
> ```

### 2.2 关联 GitHub 远程仓库并推送

1. 在 GitHub 网页上 **New repository** 建一个空仓库（如 `my-site`）。
2. 本地关联并推送：

```bash
git branch -M main                                 # 主线叫 main
git remote add origin git@github.com:你的用户名/my-site.git
git push -u origin main
```

### 2.3 日常 Git 工作流（改完代码后）

```bash
git add .
git commit -m "更新了首页文案"
git push
```

### 2.4 注意

- 本机需要配置 Git 身份：`git config --global user.email "you@x.com"` / `user.name "你的名"`。
- 如果你用 WorkBuddy 的 **GitHub 连接器**，可以免命令行做 issue/PR 等操作；但**本地 git 推送仍要在你自己的终端里做**（连接器不替代本地 git）。

## 3. 腾讯云平台需要准备哪些服务

登录 [腾讯云控制台](https://console.cloud.tencent.com)，准备以下三样：

### 3.1 服务器：轻量应用服务器（Lighthouse）

- 进入 **轻量应用服务器** → **新建实例**
- **镜像**：选 `Ubuntu 24.04 LTS`（默认登录用户是 `ubuntu`，非 root，更安全）
- **规格**：个人博客选最低配（2 核 2G 或 1 核 2G 足够）
- 创建后会拿到一个**公网 IP**（如 `115.159.67.196`），记下它

### 3.2 域名（可选但推荐）

- 在腾讯云 **域名注册** 买一个（如 `danielsdailynote.cn`，几十元/年）
- 或你已有域名，把它的 DNS 解析迁到腾讯云 DNSPod（也可留在原服务商）
- 域名和服务器**不强制同账号/同厂商**，只要你能改 DNS 记录就能用

### 3.3 SSL 证书（HTTPS 必需）

- 进入 **SSL 证书控制台** → **申请免费证书**（TrustAsia DV，单域名，有效期 **90 天**）
- 验证方式选 **DNS 验证**，按提示加一条 TXT 记录，几分钟签发
- 签发后 **下载 Nginx 格式** 压缩包，解压得到：
  - `xxx_bundle.crt`（完整证书链）
  - `xxx.key`（私钥）
- 记下证书 ID（`certId`，形如 `Z68viCQs`），后面用得上

> ⚠️ 免费证书 **90 天**到期，且**不能自动续期**——必须手动重新申请。设个提前 30 天的提醒。

### 3.4 （可选）CloudBase 兜底

- 如果你想要一个"不用管证书"的备用地址，可以在 CloudBase 开通**静态网站托管**，它会给你一个 `xxx.tcloudbaseapp.com` 的默认域名（自带 HTTPS，自动续期）。
- 但注意：**CloudBase 静态托管 2026 年起已不支持绑定自定义域名**，所以自定义域名只能走路线 A 的 Lighthouse。

## 4. 开通服务后，把项目部署到服务器

核心思路：在服务器上装 nginx，把本地 `dist/` 上传过去，配好 HTTPS。

### 4.1 用 SSH 公钥登录（不传密码，更安全）

1. 在你自己机器生成密钥对：
   ```bash
   ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N ""
   ```
2. 把 `~/.ssh/id_ed25519.pub` 的**公钥内容**追加到服务器的 `~/.ssh/authorized_keys`（可在腾讯云控制台用网页终端 OrcaTerm 粘贴；或你本地有 SSH 客户端直接 `ssh-copy-id`）。
3. 之后 `ssh ubuntu@服务器IP` 即可免密登录。

### 4.2 探测服务器环境

登录后先确认系统与权限：

```bash
whoami                 # 应显示 ubuntu（非 root）
head -3 /etc/os-release
sudo -n true && echo "免密sudo可用"   # 轻量 Ubuntu 镜像通常已配好
```

### 4.3 写 nginx 站点配置

在服务器上创建 `/etc/nginx/sites-available/yourdomain.cn.conf`：

```nginx
server {
    listen 80;
    server_name danielsdailynote.cn;
    # HTTP 强制跳 HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name danielsdailynote.cn;

    ssl_certificate     /etc/nginx/ssl/danielsdailynote.cn_bundle.crt;
    ssl_certificate_key /etc/nginx/ssl/danielsdailynote.cn.key;

    root /var/www/your-site;
    index index.html;

    # 若用 BrowserRouter 才需要下面这行；HashRouter 不要
    # location / { try_files $uri /index.html; }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

启用配置：

```bash
sudo ln -sf /etc/nginx/sites-available/yourdomain.cn.conf /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default   # 去掉默认站点
```

### 4.4 上传构建产物（⚠️ 非 root 关键坑）

`ubuntu` 用户**没有直接写** `/var/www`、`/etc/nginx/ssl` 的权限。正确做法：先传到服务器 `/tmp`（ubuntu 可写），再用 `sudo` 移动。

```bash
# 本地执行：把 dist/ 传到服务器 /tmp
scp -r dist/. ubuntu@服务器IP:/tmp/site/

# 服务器执行：sudo 移动到正式位置
sudo mkdir -p /var/www/your-site /etc/nginx/ssl
sudo cp -r /tmp/site/. /var/www/your-site/
sudo cp /tmp/danielsdailynote.cn_bundle.crt /tmp/danielsdailynote.cn.key /etc/nginx/ssl/
sudo chmod 600 /etc/nginx/ssl/*.key
```

### 4.5 安装 nginx 并启动

```bash
sudo apt-get update
sudo apt-get install -y nginx
sudo nginx -t          # 校验配置，应显示 OK
sudo systemctl enable nginx
sudo systemctl restart nginx
```

### 4.6 防火墙放行（⚠️ 轻量"防火墙" ≠ CVM"安全组"）

这是最容易搞混的一步：

- **轻量应用服务器** 在 **实例详情 → 防火墙** 标签加规则（不是 CVM 的安全组！）
- 添加两条：
  - `HTTPS(443)` 允许（必须，否则 HTTPS 打不开）
  - `HTTP(80)` 允许（用于 80→443 跳转；也可只放 443，但建议放 80 让用户访问 http 也能跳）

> 如果误进"云服务器 CVM → 安全组"，那是给 CVM 用的，对轻量实例不生效。

## 5. 把网页部署上线

### 5.1 DNS：把域名指向服务器

在你域名的 **DNS 服务商** 添加 A 记录：

- 主机记录：`@`（代表根域名）
- 类型：`A`
- 记录值：`服务器公网 IP`（如 `115.159.67.196`）
- （可选）再给 `www` 加一条同样的 A 记录

> 不知道 DNS 在哪改？看域名注册商控制台里的 **NS 记录**：
> - `*.dnspod.net` → 去腾讯云 **DNSPod** 控制台
> - `*.ns.cloudflare.com` → 去 **Cloudflare**
> - `*.alidns.com` → 去 **阿里云 DNS**

### 5.2 验证是否上线

1. **最可靠的验证**：用你**自己的浏览器**访问 `https://你的域名`。
2. 命令行验证（若环境 DNS 正常）：
   ```bash
   curl -I https://你的域名     # 应返回 200，且证书有效
   ```
3. 若命令行报"无法解析主机"，但浏览器能开——通常是**本地/沙箱 DNS 解析器连不上外网**，不是你的配置问题。此时可用强制解析绕过：
   ```bash
   curl --resolve 你的域名:443:服务器IP https://你的域名/
   ```

### 5.3 上不去时的排查顺序

1. 防火墙放 443 了吗？（轻量控制台 → 防火墙）
2. DNS A 记录生效了吗？（`nslookup 你的域名 8.8.8.8` 看是否返回服务器 IP）
3. nginx 在运行吗？（服务器上 `sudo systemctl status nginx`）
4. 证书路径对吗？（`sudo nginx -t` 会报错提示）
5. 文件真的传上去了吗？（`ls /var/www/your-site`）

## 6. 上线后如何维护

### 6.1 改内容 → 重新部署

改完本地代码后：

```bash
npm run build                                    # 重新构建 dist/
scp -r dist/. ubuntu@IP:/tmp/site/             # 重新上传
ssh ubuntu@IP "sudo cp -r /tmp/site/. /var/www/your-site/"
ssh ubuntu@IP "sudo systemctl reload nginx"      # 平滑重载，不中断服务
```

> 如果只改了文章数据（如博客的 `articles.ts`），同样重新构建 + 上传即可。

### 6.2 SSL 证书 90 天续期（重要！）

证书到期当天，浏览器会报 `NET::ERR_CERT_DATE_INVALID`，网站打不开。

1. 到期前 ~30 天，到腾讯云 **SSL 证书控制台** 重新申请一张免费证（同样 Nginx 格式）。
2. 下载后覆盖到服务器：
   ```bash
   scp 新证书_bundle.crt ubuntu@IP:/tmp/
   scp 新证书.key        ubuntu@IP:/tmp/
   ssh ubuntu@IP "sudo cp /tmp/新证书_bundle.crt /etc/nginx/ssl/danielsdailynote.cn_bundle.crt && sudo cp /tmp/新证书.key /etc/nginx/ssl/danielsdailynote.cn.key && sudo systemctl reload nginx"
   ```
3. **强烈建议设一个提前 30 天的提醒**，避免遗忘。

### 6.3 备份与回滚

- 服务器上的 `/var/www/your-site` 就是全部网站文件，定期 `scp` 拉回本地备份即可。
- 代码已在 GitHub，回滚到旧版本只需 `git checkout <旧commit>` 再重新部署。
- 可给服务器做**快照**（轻量控制台 → 快照），重大改动前拍一张。

### 6.4 监控与成本

- 轻量应用服务器是**包年包月固定费用**，留意到期续费提醒。
- 可设**缩容到 0 实例**或关机省钱（但网站会停）；个人博客建议保持运行。
- 关注流量/带宽是否超额（轻量有月流量包）。
